# Popover Component

Floating content panel anchored to a trigger element with click-outside-to-close.

---

## Overview

The Popover component displays floating content anchored to a trigger element. Features intelligent positioning, click-outside handling, and portal rendering.

**Features:**
- Intelligent positioning
- Click outside to close
- Portal rendering
- Controlled and uncontrolled modes
- ARIA attributes
- Dark mode compatible

## API Reference

### Components

- `<Popover>` - Container
- `<PopoverTrigger>` - Opens popover
- `<PopoverContent>` - Content panel

---

## Basic Usage

```razor
<Popover>
    <PopoverTrigger AsChild>
        <Button Variant="ButtonVariant.Outline">Open Popover</Button>
    </PopoverTrigger>
    <PopoverContent>
        <p>Popover content goes here</p>
    </PopoverContent>
</Popover>
```

---

## AsChild Pattern

Use `AsChild` on `PopoverTrigger` to use your own element (like a styled Button) as the trigger instead of the default `<button>`:

```razor
<Popover>
    <PopoverTrigger AsChild>
        <Button Variant="ButtonVariant.Ghost" Size="ButtonSize.Icon">
            <LucideIcon Name="settings" Size="16" />
        </Button>
    </PopoverTrigger>
    <PopoverContent>
        <p>Settings panel content</p>
    </PopoverContent>
</Popover>
```

**Note:** When `AsChild` is true, the child component (Button) automatically receives trigger behavior via `TriggerContext` including click handling, aria-haspopup, aria-expanded, and keyboard navigation.
